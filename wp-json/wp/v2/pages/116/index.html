Introduction<\/h1>\n\n

The perceptron is a linear classifier invented in 1958 by Frank Rosenblatt. It’s very well-known and often one of the first things covered in a classical machine learning course. So why create another overview of this topic?<\/p>\n\n

Well, I couldn’t find any projects online which brought together:<\/p>\n\n

    \n
  1. Visualizations of the perceptron learning in real time.<\/li>\n
  2. A proof of why the perceptron learns at all.<\/li>\n
  3. Explorations into ways to extend the default perceptron algorithm.<\/li>\n<\/ol>\n\n

    To be clear, these all exist in different places, but I wanted to put them together and create some slick visualizations with d3.<\/p>\n\n

      To get a feel for the algorithm, I’ve set up an demo below.<\/p>\n\n

      Clicking Generate Points<\/code> will pick a random hyperplane (that goes through 0, once again for simplicity) to be the ground truth. Then, points are randomly generated on both sides of the hyperplane with respective +1 or -1 labels. <\/p>\n\n

      After that, you can click Fit Perceptron<\/code> to fit the model for the data. You can see each misclassified point flash briefly, moving the perceptron’s weights either up or down, respectively throughout the training procedure.<\/p>\n\n

      At each iteration of the algorithm, you can see the current slope of w<\/mi>t<\/mi><\/msub><\/mrow>w_t<\/annotation><\/semantics><\/math><\/span><\/span>w<\/span><\/span>t<\/span><\/span><\/span><\/span>&ZeroWidthSpace;<\/span><\/span><\/span><\/span><\/span><\/span><\/span><\/span><\/span><\/span><\/span><\/span> as well as its error on the data points.<\/p>\n\n

      Because all of the data generated are linearly separable, the end error should always be 0. However, note that the learned slope will still differ from the true slope!
